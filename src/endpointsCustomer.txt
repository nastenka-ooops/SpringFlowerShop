Customer API Endpoints Documentation

## 1. Получение информации о всех клиентах
- **Описание**: Этот эндпоинт позволяет получить информацию о всех клиентах магазина.
- **Путь**: `GET /api/v1/customers`
- **Параметры**: нет
- **Ответ в случае успеха**: Список объектов `CustomerDto`.

## 2. Получение информации о клиенте по идентификатору
- **Описание**: Этот эндпоинт позволяет получить информацию о конкретном клиенте по его идентификатору.
- **Путь**: `GET /api/v1/customers/{id}`
- **Параметры**:
  - `id` (число, обязательный) - идентификатор клиента
- **Ответ в случае успеха**: Объект `CustomerDto`.
- **Ответ в случае, если клиент не найден**: Статус 404 "Not Found".

## 3. Получение информации о заказах клиента по его идентификатору
- **Описание**: Этот эндпоинт позволяет получить информацию о всех заказах клиента по его идентификатору.
- **Путь**: `GET /api/v1/customers/{id}/orders`
- **Параметры**:
  - `id` (число, обязательный) - идентификатор клиента
- **Ответ в случае успеха**: Список объектов `OrderDto`.
- **Ответ в случае, если клиент не найден**: Статус 404 "Not Found".

## 4. Создание нового клиента
- **Описание**: Этот эндпоинт позволяет создать нового клиента в магазине.
- **Путь**: `POST /api/v1/customers/{id}`
- **Параметры**:
  - `id` (число, обязательный) - идентификатор клиента
- **RequestBody**:
  - `first_name` (строка, обязательный) - имя клиента
  - `last_name` (строка, обязательный) - фамилия клиента
  - `email` (строка, обязательный) - адрес электронной почты клиента
  - `address` (строка, обязательный) - адрес клиента
  - `phone` (строка, обязательный) - номер клиента
- **Ответ в случае успеха**: Объект `CustomerDto`.

## 5. Обновление данных клиента по идентификатору
- **Описание**: Этот эндпоинт позволяет изменить данные существующего клиента по его идентификатору.
- **Путь**: `PUT /api/v1/customers/{id}`
- **Параметры**:
  - `id` (число, обязательный) - идентификатор клиента
- **RequestBody**:
  - `first_name` (строка, обязательный) - имя клиента
    - `last_name` (строка, обязательный) - фамилия клиента
    - `email` (строка, обязательный) - адрес электронной почты клиента
    - `address` (строка, обязательный) - адрес клиента
    - `phone` (строка, обязательный) - номер клиента
- **Ответ в случае успеха**: Объект `CustomerDto`.
- **Ответ в случае, если клиент не найден**: Статус 404 "Not Found".

## 6. Удаление клиента по идентификатору
- **Описание**: Этот эндпоинт позволяет удалить клиента из магазина по его идентификатору.
- **Путь**: `DELETE /api/v1/customers/{id}`
- **Параметры**:
  - `id` (число, обязательный) - идентификатор клиента
- **Ответ в случае успеха**: Статус 204 "No Content".
- **Ответ в случае, если клиент не найден**: Статус 404 "Not Found".
